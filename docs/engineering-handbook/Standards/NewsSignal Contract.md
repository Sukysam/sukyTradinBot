# Standard — NewsSignal Contract

Governs `nlp.models.NewsSignal`, the single output type the NLP & Event
Processing layer (Milestone 10) is expected to produce. See
[Architecture/ADR/ADR-018-NewsSignal-Contract.md](../Architecture/ADR/ADR-018-NewsSignal-Contract.md)
for why this type is frozen *before* `src/nlp/` exists at all —
following the same "freeze interfaces before implementation" discipline
[ADR-016](../Architecture/ADR/ADR-016-LearningDecision-Contract.md)
established for `ExperienceRecord`/`LearningDecision`. This document is
the binding contract for whoever implements the NLP layer against it, and
for every future consumer (Signal Orchestration, Milestone 11).

## Why this exists, and why now

Milestone 10's mandate, per direct product-owner review, deliberately
narrows the uploaded architecture documents' original sequencing (FinBERT
sentiment integrated directly into trading logic): **shadow mode again,
and infrastructure before the model.** `NewsSignal` is recorded for every
processed story; the Strategy Engine remains completely unaware of it
during this milestone, the same "opinion recorded, never consumed"
boundary [ADR-016](../Architecture/ADR/ADR-016-LearningDecision-Contract.md)
already established for `LearningDecision`. A real reference
implementation already exists — `regime-trader/core/sentiment_engine.py`
(`SentimentEngine`, FinBERT-backed, `SentimentScore`) and
`regime-trader/broker/news_streamer.py` (`NewsStreamer`, `NewsItem`) — so,
per [05_MEMORY_ENGINEER.md](../05_MEMORY_ENGINEER.md)/ADR-016's precedent
for the Memory Loop, this contract **adapts** that validated shape rather
than inventing one from scratch or porting it unexamined.

`NewsSignal` is deliberately the *only* frozen output of this milestone.
The internal pipeline stages that produce it — ingestion, cleaning,
deduplication — are explicitly **not** frozen, the same way
[ADR-013](../Architecture/ADR/ADR-013-Execution-Layer-Design.md) left
`ExecutionContext`/`FeatureSnapshot` as internal, unfrozen value objects:
those stages are implementation detail Milestone 10 is free to design and
iterate on, because nothing downstream is meant to depend on their shape
directly.

## Scope

Applies to `nlp.models.NewsSignal` and whatever public method the
eventual `nlp.service` module exposes to produce it (signature not yet
fixed — that's implementation, not contract). Does **not** freeze: how
raw news events are ingested or transported (the legacy `NewsStreamer`/
`NewsItem` pattern is a reasonable starting point, not mandated), how
deduplication is performed, or the entity-extraction method used — all
Milestone 10 implementation detail. Does not freeze a phasing schedule
either, though ADR-018 records the phased build order actually used.

## Required fields

| Field | Type | Guarantee |
|---|---|---|
| `signal_id` | `str` | Never empty. Identifies this `NewsSignal` itself — one per de-duplicated, fully-processed story, not necessarily equal to the originating feed's own story ID. |
| `source_id` | `str` | Never empty. Traceability back to the raw feed event this signal was derived from — the legacy `NewsItem.id` equivalent, carried as `str` rather than `int` so a future non-Alpaca source isn't forced into an integer ID scheme. |
| `source` | `str` | Never empty. Which feed/provider produced the raw event (e.g. `"alpaca_news"`). |
| `symbols` | `tuple[str, ...]` | May be empty — some macro/market-wide stories aren't tagged to a specific ticker by the feed itself. Feed-tagged, not NLP-derived; mirrors the legacy `NewsItem.symbols`. Every non-empty entry must itself be non-empty (no blank-string tickers). |
| `entities` | `tuple[str, ...]` | May be empty — entity extraction may find nothing, or may run before any implementation exists to populate it meaningfully. NLP-derived, independent of `symbols`: no rule requires overlap or de-duplication between the two, since one is feed metadata and the other is a model's own extraction. Every non-empty entry must itself be non-empty. |
| `headline` | `str` | Never empty. The text sentiment was actually scored against — this contract follows the reference engine's own calibration (`06_NLP_ENGINEER.md`'s documented pitfall: FinBERT here is calibrated against headlines specifically, not `summary`). |
| `published_at` | `datetime` | Timezone-aware, normalized to UTC. When the source published the story. |
| `processed_at` | `datetime` | Timezone-aware, normalized to UTC. When this `NewsSignal` was generated by the pipeline. Must be `>= published_at` — causality, the same "no look-ahead" spirit invariant #1 already requires elsewhere, applied to a signal that can only be computed after its source material exists. |
| `sentiment_positive` | `float` | `[0.0, 1.0]`. |
| `sentiment_negative` | `float` | `[0.0, 1.0]`. |
| `sentiment_neutral` | `float` | `[0.0, 1.0]`. |
| `sentiment_label` | `str` | One of exactly `{"positive", "negative", "neutral"}` — the same `REQUIRED_LABELS` set `regime-trader/core/sentiment_engine.py` already validates against at model-load time. Must equal the argmax of `sentiment_positive`/`sentiment_negative`/`sentiment_neutral` — validated at construction, extending "the label already always matches the highest score" from an implementation detail (true only because the reference engine always computes it that way) into a type-level guarantee no caller can violate. |
| `model_version` | `str` | Never empty. Identifies the sentiment-model version that produced this signal — same traceability role `RegimeState.model_version`/`LearningDecision.model_version` already play. A model swap (e.g. a different FinBERT checkpoint) is visible on every signal it produced, not silently invisible. |
| `metadata` | `Mapping[str, Any]` | Free-form. **No guaranteed keys yet** — no implementation exists yet to honestly commit to specific keys, the same reasoning [ADR-008](../Architecture/ADR/ADR-008-StrategyDecision-Contract.md) applied to `StrategyDecision.metadata`. A caller that wants to carry `summary` or other ingestion-time text through can do so here without it being a contract promise. |

The three sentiment probabilities must sum to within `[0.99, 1.01]` —
the exact tolerance `regime-trader/core/sentiment_engine.py`'s
`SentimentScore.__post_init__` already uses, kept identical rather than
re-derived differently here, since it reflects real observed
floating-point softmax behavior, not an arbitrary choice.

`NewsSignal` must be an immutable (`frozen=True`) dataclass, matching
every other decision/signal-shaped contract in this handbook.

## The shadow-mode guarantee

Stated explicitly, the same way [ADR-016](../Architecture/ADR/ADR-016-LearningDecision-Contract.md)'s
Standards doc states it for `LearningDecision`: **no code path in this
milestone constructs a `StrategyDecision`, `ExecutionDecision`, or
`OrderIntent` from a `NewsSignal`.** The Strategy Engine gains no new
dependency on `nlp` in Milestone 10 — `NewsSignal`s are recorded (however
Milestone 10's implementation chooses to persist them) and nowhere else.
Wiring `NewsSignal` into a real trading decision is Milestone 11's job
(Signal Orchestration), consuming `StrategyDecision` + `LearningDecision`
+ `NewsSignal` together — a deliberate, later, separately-designed
convergence point, not something this contract's existence already
authorizes.

## Versioning policy

Follows the same three-tier pattern as every prior contract in this
handbook (see
[StrategyDecision Contract.md](StrategyDecision%20Contract.md#versioning-policy)):
a contract-shape version (this document's own "Contract history" below)
is independent of whatever internal versioning the eventual sentiment
model or entity extractor defines for itself (`model_version`). Currently
**v1** (this freeze, ADR-018) — no implementation exists yet to have
driven a v2.

## Backward compatibility expectations

Same allowed/requires-a-new-ADR/never-permitted structure as
[RegimeState Contract.md](RegimeState%20Contract.md#backward-compatibility-expectations),
applied to `NewsSignal`. Notably: adding guaranteed `metadata` keys once
a real implementation exists is additive; widening `sentiment_label`'s
required set beyond `{"positive", "negative", "neutral"}` (e.g. a model
swap to a finer-grained label scheme) or allowing `NewsSignal` to be
consumed by `strategy`/`risk`/`execution` would each require a new ADR
and explicit [01_SYSTEM_ARCHITECT.md](../01_SYSTEM_ARCHITECT.md)
sign-off — the second of those in particular is exactly the kind of
decision this contract's shadow-mode guarantee exists to make
deliberate, not incidental.

## Contract history

- **v1** ([ADR-018](../Architecture/ADR/ADR-018-NewsSignal-Contract.md)):
  initial freeze — `signal_id`, `source_id`, `source`, `symbols`,
  `entities`, `headline`, `published_at`, `processed_at`,
  `sentiment_positive`, `sentiment_negative`, `sentiment_neutral`,
  `sentiment_label`, `model_version`, `metadata`. No implementation
  exists yet; this is the contract Milestone 10 is built against, not a
  retrofit onto existing code.

## Enforcement

Not yet mechanically enforced — there is no `nlp.models` module yet. The
first implementation ships `tests/nlp/test_models.py` enforcing every
constraint in the Required Fields table above (frozen dataclass, UTC
timestamps, `processed_at >= published_at`, non-empty string fields,
every `symbols`/`entities` entry non-empty, sentiment probabilities in
`[0.0, 1.0]` summing to `[0.99, 1.01]`, `sentiment_label` in the required
set and consistent with the argmax), in the same change that adds the
module, matching every other contract in this handbook.

## Ownership

Build and maintain: [06 NLP Engineer](../06_NLP_ENGINEER.md), per that
role's existing mandate over headline sentiment scoring — extended here
to own the full `NewsSignal` production pipeline (ingestion through
signal), not just the scoring step alone. A consumer that needs a
capability this contract doesn't provide — in particular, anything that
would let a `NewsSignal` influence a real trade before Milestone 11 —
raises it against this document and
[01_SYSTEM_ARCHITECT.md](../01_SYSTEM_ARCHITECT.md) explicitly. It does
not reach into `nlp` internals, or into `strategy`/`risk`/`execution`, to
wire that up unreviewed.
